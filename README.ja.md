# GifDance

[English](README.md) | 日本語

<img width="1920" height="1080" alt="Image" src="https://github.com/user-attachments/assets/60dc36b8-3e1a-45a7-a5df-6db97547d022" />
`GifDance` は、ローカルの GIF を読み込み、ホストのトランスポートに同期して再生する Windows 用 VST3 エフェクトプラグインです。

## 挙動

- 音声処理はパススルーのみです。
- エディタ全体を GIF の表示領域として使います。
- マウスホバーで歯車アイコンが表示され、クリックするとファイル読込や再生設定用のオーバーレイが開きます。
- GIF の再生は、ホストの `PPQ` 位置に追従します。
- `Loop Beats` はオートメーション可能で、最小 `1/8` 拍までの短いループに対応しています。
- `Start frame` と `End frame` で、GIF の一部分だけをループ対象にできます。
- `Offset frame` で、選択範囲のどのフレームからループを始めるかをずらせます。
- `Ping Pong` モードを有効にすると、選択範囲の終端まで進んだあと、先頭へ戻らず逆方向に再生します。
- ホスト停止中は、選択したオフセット位置のフレームを表示します。
- プラグインウィンドウのリサイズ時も、GIF はアスペクト比を維持したまま表示されます。
- プラグインウィンドウのサイズは state に保存され、プロジェクト再読込時に復元されます。
- GIF デコードは透明を含む合成済みフレームに対応しています。
- プラグイン state には、GIF の絶対パス、選択したループ長、フレーム範囲、再生モード、エディタサイズが保存されます。

## 使い方

1. REAPER のトラックに `GifDance` を挿します。
2. プラグインウィンドウ上にマウスを乗せ、歯車アイコンをクリックします。
3. `Open GIF` を押して、ローカルの `.gif` ファイルを選びます。
4. `Loop Beats` で、音楽的なループ長を設定します。
5. GIF の一部分だけを使いたい場合は、`Start frame` と `End frame` を調整します。
6. その範囲の中でループ開始位置をずらしたい場合は、`Offset frame` を設定します。
7. 終端で折り返す再生にしたい場合は、`Ping Pong` を有効にします。
8. 必要に応じてプラグインウィンドウをリサイズします。GIF は縦横比を維持したまま収まります。

## 状態保存と同期

- 再生はフリーランのタイマーではなく、ホストのトランスポート位置を基準に決まります。
- シーク、再生再開、テンポ変更を行っても、その時点の拍位置へ即座に再整列します。
- 保存されていた GIF パスが再読込時に見つからない場合も、クラッシュせず、パスを保持したままエラー表示になります。

## ビルド

このプロジェクトは JUCE を同梱しており、CMake で構成します。

```powershell
cmake -S . -B build-vs -G "Visual Studio 17 2022" -A x64
cmake --build build-vs --config Release --target GifDance_VST3
```

ビルドされたプラグインの出力先:

`build-vs/GifDance_artefacts/Release/VST3/GifDance.vst3/Contents/x86_64-win/GifDance.vst3`

## メモ

- この環境では REAPER は検出されていますが、ビルド時にグローバルな VST3 フォルダへ自動インストールはしません。
- ローカルの `build/` ディレクトリには、途中で止めた Ninja 構成の残りが入っている可能性があります。検証済みの出力は `build-vs/` です。
